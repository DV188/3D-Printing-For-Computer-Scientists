A Programmers Option

If the command line is where you feel most comfortable as a computer scientist with former programming experience, feel free to download OpenSCAD using your package manager of choice. On Mac OS X, using both Homebrew or MacPorts should provide the user with the same program discussed in Getting Started.

$ brew install openscad

Different Linux distributions will also come with package managers that can be used to the same affect:

$ sudo apt-get install openscad

Edit As You Like

One neat trick that can improve your coding in OpenSCAD is using the text editor you are familiar with instead of the default. OpenSCAD's text editor is fairly basic and limited in what it is able to achieve. Open the program, navigate to Design > Automatic Reload and Preview, make sure this option is check marked. What this allows us to do is re-render our models by simply saving the working OpenSCAD file. Using this in conjunction with an OpenSCAD alias can increase your productivity in OpenSCAD using software you are comfortable with and use already.

$ alias openscad=/Applications/OpenSCAD.app/Contents/MacOS/OpenSCAD

The file extension OpenSCAD uses is .scad. Using the editor of your choice (vim, emacs, notepad++, etc), opening up .scad files, making changes, and saving the changes, will automatically refresh the rendered image being displayed in the OpenSCAD software. Useful plugins can also be found such as vim-openscad and many others, allowing you further control over what you are programming.
