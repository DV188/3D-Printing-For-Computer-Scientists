External Files

Until this point, all our OpenSCAD models have been contained in a single file. OpenSCAD allows programmers to import external libraries and files to be used together.

Include And Use

The include and use functions draw from external files. Include acts as a preprocessor and copies the code specified from one file to the current working file. The use function imports the modules and functions but does not run the code unless you call the functions explicitly.

include_and_use.scad
 
/*
 * Include and use draw code from another external file for use within the calling code.
 */

include  // Draws the flag as if it were pasted into this file.

use  // Allows us to use the functions written in the file.
echo(fibonacci(10));

Using the functions created in the Functions And Modules page by including and using what was previously written.

Creating .stl Files

With your model finally completed, the last step in the CAD portion of creating a 3D printable object is saving you file as an .stl. STL stands for stereolithography. Nearly all consumer 3D printers used today have the .stl file format as the default standard.

Using OpenSCAD, negotiate to the File > Export > Export as STL... option to export your model to a printable format.

You have now created both a project file, .scad, and a file ready to be converted into G-code, .stl.

Import .stl Files

Once you have created your .stl file, there are many different software packages that will allow you to view the STL such as Cura or Repetier. But, OpenSCAD does offer the import function that imports an .stl to be viewed from within the program.

import_stl.scad
 
/*
 * The import function is used to import .stl file typically created with CAD software.
 */
import("/path/to/file.stl");
